# Picture Book — offline tap-to-read

A single Flutter codebase that runs on **iOS, iPad, Android, macOS, and
Windows**. Kids tap a sentence to hear it read aloud, or tap a word to hear
it and see its Chinese translation + English definition. Teachers manage
books behind a PIN gate. Every byte of TTS audio, every image, and every
dictionary entry stays on-device — the app never touches the network at
runtime.

## Features

- **Reader with two modes** — *Word by word* (tap a word → speak it + meaning
  sheet with translation, pinyin, and example sentences) and *Whole sentence*
  (tap anywhere → speak the sentence + show its Chinese translation).
- **Two TTS engines** — the platform's system voice, plus an optional bundled
  **Kokoro neural voice** (via `sherpa_onnx`). `TtsRouter` picks per speak
  call based on the user's Settings toggle and whether the model is bundled.
- **Bundled English→Chinese dictionary** — ECDICT-based SQLite DB shipped as
  an asset, opened read-only. Includes a suffix-stripping stemmer so "days"
  resolves to "day".
- **PDF import** — renders each PDF page to JPEG, extracts embedded text, and
  falls back to on-device **Apple Vision OCR** on iOS/macOS. Auto-fills word
  translations from the dictionary.
- **Seed library** — first launch imports the bundled Oxford reading-tree
  books (baked as `.book.zip` files; see below).
- **Offline license activation** — Ed25519-signed license codes generated by
  a developer-side tool and verified in-app against an embedded public key.
  No activation server.
- **Teacher admin** behind a PIN gate: book list, page editor, PDF import,
  seed baking, backup/restore via ZIP.

## Quick start (for collaborators)

The repo includes the platform folders (`ios/`, `android/`, `macos/`,
`windows/`), so it runs straight after cloning — no `flutter create` step.

```sh
git clone https://github.com/kappaborg/tutoringApp.git
cd tutoringApp
flutter pub get
flutter test                 # 84 tests, all should pass
flutter run -d <device_id>   # `flutter devices` lists targets
```

Requires a recent stable Flutter SDK (Dart ≥ 3.4).

### Full setup on a fresh machine

To restore the complete app — including the two large asset sets that are
not in git (see below) — the whole sequence is:

```sh
git clone https://github.com/kappaborg/tutoringApp.git
cp -R ~/Backups/picturebook-seed/oxford/* tutoringApp/assets/seed/oxford/
cd tutoringApp
dart run tool/fetch_kokoro.dart    # re-downloads the neural voice (~350 MB)
flutter pub get && flutter test
```

The `cp` step assumes the baked Oxford library backup at
`~/Backups/picturebook-seed/oxford/` (166 `.book.zip` files, ~2.3 GB) —
that backup lives on the maintainer's machine. Contributors who don't have
it should ask the maintainer (@kappaborg) for a copy, or re-bake the set
from the source PDFs as described below. The app still runs without it;
first launch just skips the seed import.

### Large assets (not in git)

Two asset sets are gitignored because of their size. The app runs without
them — neural TTS silently falls back to the system voice, and the seed
import is skipped — but a release build needs both.

| Asset | Size | How to get it |
|---|---|---|
| `assets/voices/kokoro/` — Kokoro neural TTS model | ~350 MB on disk | `dart run tool/fetch_kokoro.dart` (one-time download from the sherpa-onnx releases, Apache-2.0) |
| `assets/seed/oxford/*.book.zip` — bundled book library | ~2.3 GB (166 books) | Copy from the maintainer's backup (see above), or bake from source PDFs on macOS: `flutter run -d macos` → Admin → **Bake seed library** → pick the PDF folder. See `assets/seed/oxford/README.md`. |

Both are build-time steps on a developer machine; the shipped app never
downloads anything.

### Full dictionary (optional, recommended)

`assets/dict/ecdict.db` is committed but ships as a small starter set. To
upgrade it to the full ECDICT lite (~70k entries):

```sh
dart run tool/fetch_ecdict.dart      # download + rebuild in one command
git add assets/dict/ecdict.db        # commit the regenerated asset
```

## License activation workflow

Customer licenses are offline, signed codes of the form
`<base64url(payload)>.<base64url(ed25519_signature)>`.

```sh
# One-time project setup — writes the private key OUTSIDE the repo
# (~/.picturebook/private_key.bin). Never commit or share this file.
dart run tool/generate_keypair.dart

# Issue a code for a customer (optionally render it as a QR PNG):
dart run tool/issue_license.dart --customer="Wang Mei" --tier=full --qr=/tmp/wang.png
```

The matching public key is embedded in
`lib/services/license_service.dart`; the app verifies codes locally on the
activation screen. Rotating the keypair invalidates every issued license —
only do it after a key compromise.

## Development

```sh
flutter analyze                      # static analysis (should be clean)
flutter test                         # unit + widget tests
dart run tool/check_offline.dart     # offline-guard: bans network deps/APIs
```

### Airplane-mode verification protocol

1. Turn off Wi-Fi, cellular, Bluetooth (all radios off).
2. Cold-launch the app: the seeded books render.
3. Tap a sentence → TTS speaks it. Tap a word → meaning popup + speech.
4. Set a Teacher PIN, create a book with a new image, save.
5. Export a backup ZIP, import it back — counts match.

Anything reaching for the network is a regression — re-run
`dart run tool/check_offline.dart`.

## Offline guarantees

1. **No banned dependencies** in `pubspec.yaml` — guard list in
   `tool/check_offline.dart`.
2. **No banned APIs** in `lib/` (no `HttpClient`, `Socket.connect`,
   `WebSocket.connect`, `dart:html`). The single allowlisted file,
   `lib/services/debug_network_probe.dart`, is debug-only.
3. **Android** manifest removes any transitive `INTERNET` permission with
   `tools:node="remove"`.
4. **macOS** entitlements set `network.client`/`network.server` to `false`.
5. **iOS** disables arbitrary loads via `NSAppTransportSecurity`.
6. CI script + `test/offline_guard_test.dart` enforce all of the above.

## Known platform caveats

| Platform | Caveat |
|---|---|
| iOS Simulator | No TTS voices by default — install one in **Settings → Accessibility → Spoken Content → Voices**, or test on a real device. |
| Windows | System TTS uses SAPI (install voices in **Settings → Time & language → Speech**); no word-progress callbacks, so highlighting falls back to whole-sentence. |
| macOS | App is sandboxed — the file picker re-prompts after every launch; that's sandbox behavior, not a bug. Seed baking is macOS-only. |
| Android < 13 | Uses `READ_EXTERNAL_STORAGE`; 13+ uses `READ_MEDIA_IMAGES`. The manifest handles both. |
